一、项目背景
================================================================================

  在现代软件开发过程中，代码质量是保障软件可靠性和可维护性的关键因素。
  然而，人工代码审查效率低下且容易遗漏问题。本系统针对这一痛点，提供了
  一套自动化的代码质量检测解决方案，能够快速识别Java代码中的命名不规范、
  安全漏洞、代码异味等问题，并给出评分和修改建议。

三、核心功能设计
================================================================================

  3.1 代码检测引擎
  引擎接收源代码文本，按行分割后逐条匹配规则，生成问题报告。

  检测流程：
  ┌─────────┐    ┌───────────┐    ┌───────────┐    ┌──────────┐
  │ 接收代码 │ -> │ 解析文件  │ -> │ 逐规则    │ -> │ 生成评分 │
  │          │    │ 获取规则  │    │ 扫描匹配  │    │ 和报告   │
  └─────────┘    └───────────┘    └───────────┘    └──────────┘

  3.2 评分算法
  基础分100分，根据问题严重程度递减：
  score = 100 - (criticalCount * 10) - (majorCount * 5) - (minorCount * 2)
  最低0分，最高100分。

  3.3 内存存储
  所有数据存储于 ConcurrentHashMap 中，无需安装数据库。
  · CodeCheckRecordMapper：检测记录存储
  · CodeIssueMapper：检测问题存储  
  · CheckRuleMapper：检测规则存储（预置12条规则）
  · SystemConfigMapper：系统配置存储

================================================================================

  · 单次检测响应时间：< 1秒（取决于代码长度）
  · 最大支持文件：不限
  · 最大支持单文件行数：不限
  · 吞吐量：支持并发检测

五、安全说明
================================================================================

  · 系统不连接外部网络，代码仅在本地检测
  · 无数据库，无用户数据泄露风险
  · CORS 已配置允许跨域访问
  · 检测规则包含安全检测项（禁止System.exit、禁止eval等）
